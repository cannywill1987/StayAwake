# 桌面组件实现方案

## 组件类型

| 平台 | 类型 | 是否涉及 | 入口文件 | 说明 |
|---|---|---:|---|---|
| macOS | Status Bar | 是 | `macos/Runner/AppDelegate.swift` | 原生菜单栏入口 |
| Flutter macOS | 主窗口 | 是 | `lib/main.dart` | 用户控制台 |
| Android/iOS/Windows | 系统小组件 | 否 | 不涉及 | MVP 不覆盖 |

## 数据快照

| 字段 | 来源 | 缓存位置 | 刷新时机 | 是否敏感 |
|---|---|---|---|---|
| active | Native assertion | 内存 | start/stop/getStatus | 否 |
| endsAt | Flutter/native | 内存 | session 创建 | 否 |
| custom duration hours/minutes | Status Bar custom view | 内存 | 用户在 `持续` 面板输入 | 否 |
| until clock time | Status Bar custom view | 内存 | 用户在 `至` 面板选择或拖动表盘 | 否 |
| preventDisplaySleep | Flutter switch | 内存 | startSession | 否 |
| allowScreenSaver | Flutter switch | 内存 | startSession | 否 |
| power source | macOS IOKit.ps | 不缓存 | getPowerStatus / 30s tick | 否 |
| batteryPercent | macOS IOKit.ps | 不缓存 | getPowerStatus / 30s tick | 否 |
| settings/rules/history | Flutter local JSON | Application Support | 用户修改 / session 事件 | 否 |

## Bridge Action

| action | 入参 | 返回 | 原生端 | Flutter 回流 |
|---|---|---|---|---|
| `startSession` | durationSeconds, preventDisplaySleep, allowScreenSaver | status payload | 创建 IOKit assertion | `nativeStatusChanged` |
| `stopSession` | 无 | status payload | release assertion | `nativeStatusChanged` |
| `getStatus` | 无 | status payload | 读取当前状态 | 无 |
| `startPreset` | seconds | 无 | 菜单栏点击 | Flutter 同步启动 |
| `getPowerStatus` | 无 | power payload | 读取电源和电量 | Flutter Rules / Status 页面刷新 |
| `getFrontmostApp` | 无 | app payload | 读取前台 App | Flutter Rules 页面刷新 |
| `syncPreferences` | settings, rules | status payload | 同步菜单栏选中态和自定义时长 | 无 |

## 本轮实现补充

- 菜单栏入口使用 `STAY` / `STAY ON` 文本，确保在用户菜单栏图标较多时仍可发现。
- 菜单栏提供 Amphetamine 风格中文 NSMenu：`开启新会话:`、`无限期`、`分钟`、`小时`、`自定义时间 / 直到`、`当 App 正在运行时`、`当下载文件时...`、`快速设置`、`设置...`、`关于 StayAwake`、`反馈和支持`、`关闭 StayAwake`。
- `自定义时间 / 直到` 使用原生 `NSMenuItem.view` 承载 `CustomTimeMenuView`，包含 `持续 / 至` 分段、小时/分钟输入、步进器、`NSDatePicker`、可拖动 `CustomClockView` 和 `继续` 按钮。
- `继续` 会将持续时长或直到目标时间换算成 seconds，并复用 `startPreset` bridge 启动 session。
- 菜单栏 start/stop 优先回调 Flutter，由 Flutter 再统一调用 `startSession` / `stopSession`，避免 native 与 Flutter 重复创建 session。
- Flutter 左侧导航已覆盖 Status、Sessions、Rules、Settings 四个真实页面。
- Rules 页面新增插电自动开启、低电量自动停止、前台 App 触发、下载文件触发规则。
- Settings 和 session history 保存到本机 `~/Library/Application Support/StayAwake/state.json`。
- `getPowerStatus` 返回 `available / source / isPluggedIn / batteryPercent`，用于 UI 展示和规则判断。
- `getFrontmostApp` 返回 `available / name / bundleIdentifier`，用于选择 App trigger 目标。

## 降级与风险

- 菜单栏仍使用原生 NSMenu；`自定义时间 / 直到` 已使用 custom view 面板，复杂全局设置仍在 Flutter 主窗口完成。
- `allowScreenSaver` 当前作为 session policy 字段展示和持久化，未直接修改系统屏保偏好。
- Login Item 未接入 native ServiceManagement，本轮仅保存 `startAtLogin` 偏好。
